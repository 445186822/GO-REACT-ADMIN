最终全栈组合说明

项目定位：
轻量、现代、企业级可复制、可扩展、AI-ready 的后台管理 demo/template。


一、后端技术栈

1. Go
作用：
- 后端主语言。
- 编译快、启动快、部署轻量。
- 适合 AI 修改代码后的快速验证。
- 适合构建轻量企业服务、API 平台、WebSocket/SSE 实时服务。

2. Echo
作用：
- Go Web API 框架。
- 负责 HTTP 路由、中间件、请求绑定、统一错误处理。
- Handler 统一返回 error，适合做企业级错误收口。
- 比较适合构建风格统一、可维护的现代 Go API。

3. SQLC
作用：
- 从 SQL 生成类型安全的 Go 数据访问代码。
- 避免 ORM 魔法，让 SQL 显式、可控、可审查。
- 数据库字段、参数类型、返回结构在编译期暴露问题。
- 适合严谨、长期维护的企业级数据访问层。

4. PostgreSQL
作用：
- 主业务数据库。
- 支持事务、复杂查询、JSON、索引、视图、Row Level Security 等企业级能力。
- 适合承载用户、角色、权限、订单、审批、日志等核心数据。

5. Redis
作用：
- 缓存、限流、验证码、Token 黑名单、系统配置缓存、实时通知辅助。
- 可用于 WebSocket 在线状态、未读消息计数等场景。

6. OpenAPI
作用：
- 前后端接口契约。
- 后端定义接口规范，前端生成 TypeScript API client。
- 减少手写接口类型，降低前后端字段不一致风险。
- 适合企业项目长期维护和多人协作。

7. JWT + Refresh Token
作用：
- 登录认证机制。
- JWT 用于接口访问认证。
- Refresh Token 用于无感刷新登录状态。
- 适合前后端分离后台系统。

8. RBAC
作用：
- 角色权限模型。
- 支持用户、角色、菜单、按钮权限。
- 典型权限码示例：user:view、user:create、order:export、approval:approve。

9. Data Scope
作用：
- 数据权限。
- 控制不同用户可访问的数据范围。
- 示例：超级管理员看全部，部门主管看本部门，普通员工只看本人创建的数据。

10. WebSocket
作用：
- 实时双向通信。
- 用于系统通知、审批提醒、任务提醒、在线状态、未读消息实时更新。

11. SSE
作用：
- Server-Sent Events，服务端流式输出。
- 用于 AI 助手流式返回、长任务进度推送。
- 比 WebSocket 更适合单向流式文本输出场景。

12. golang-migrate
作用：
- 数据库迁移工具。
- 管理表结构变更。
- 让数据库结构和代码版本一起演进。

13. slog / zap
作用：
- 后端结构化日志。
- 记录请求日志、错误日志、审计日志、trace id。
- 方便企业项目排查问题。

14. Docker Compose
作用：
- 本地和 demo 环境一键启动。
- 编排后端、前端、PostgreSQL、Redis 等服务。
- 降低部署和体验成本。


二、前端技术栈

1. React
作用：
- 前端主框架。
- 组件就是函数，UI 用 JSX/TSX 表达。
- 灵活，适合动态表单、动态表格、权限驱动 UI、低代码式页面配置。

2. TypeScript
作用：
- 前端类型系统。
- 配合 OpenAPI 生成的 API 类型，提高接口调用可靠性。
- 适合企业项目长期维护和重构。

3. Vite
作用：
- 前端构建和开发服务器。
- 启动快、热更新快。
- 适合 AI 修改前端代码后的快速验证。

4. Ant Design
作用：
- React 企业级 UI 组件库。
- 提供表格、表单、弹窗、菜单、布局、上传、树、日期选择等基础组件。
- 适合后台管理系统。

5. Ant Design ProComponents
作用：
- Ant Design 生态中的中后台增强组件。
- 重点使用 ProTable、ProForm、ProDescriptions。
- 适合快速构建企业 CRUD、查询表格、复杂表单、详情页。

6. React Router
作用：
- 前端路由。
- 管理登录页、后台布局页、业务页面。
- 配合权限实现路由守卫和动态菜单。

7. TanStack Query
作用：
- 服务端状态管理。
- 管理列表、详情、分页、缓存、刷新、重试、mutation。
- 普通业务数据不放全局 store，而是交给 TanStack Query。
- 非常适合 CRUD 后台。

8. Zustand
作用：
- 轻量全局状态管理。
- 存放登录用户、权限码、菜单、主题、全局 UI 状态。
- 不负责接口列表数据，避免全局状态膨胀。

9. OpenAPI generated client
作用：
- 根据后端 OpenAPI 自动生成 TypeScript 请求客户端。
- 前端不手写接口路径和返回类型。
- 后端接口变更后，前端可以通过类型错误及时发现问题。

10. Vitest
作用：
- 前端单元测试和组件测试。
- 测试工具函数、hooks、权限组件、关键业务组件。

11. Playwright
作用：
- 端到端测试。
- 验证登录、CRUD、权限控制、WebSocket 通知、AI 流式页面等核心流程。

四、最终组合一句话

Backend:
Go + Echo + SQLC + PostgreSQL + Redis + OpenAPI + JWT/RBAC/DataScope + WebSocket + SSE

Frontend:
React + TypeScript + Vite + Ant Design + ProComponents + React Router + TanStack Query + Zustand + OpenAPI Client

目标：
不是一次性 demo，而是可以直接复制到企业后续项目中的轻量级全栈模板。
